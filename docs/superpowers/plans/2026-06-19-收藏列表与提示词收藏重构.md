# 收藏列表与提示词收藏重构 Implementation Plan

> **For agentic workers:** inline 执行（后端 TDD pytest + 前端 vitest/vue-tsc/build）。

**Goal:** 把「收藏列表」拆为独立页签（带搜索+跳转），把「提示词收藏」页重构为图库网格工作台（本页上传反推 + 拆分），反推图存 promptbox 工作区与图库隔离。

**Architecture:** 后端新增 promptbox workspace 临时存储 + `/api/promptbox/analyze` 反推端点（复用 get_tagger + classifier.classify，不碰图库）。前端新建 CollectionListPage（搜索+网格+跳转），重构 PromptBoxPage 为上传反推→网格→拆分工作台，复用图库 ImageCard/n-grid/useTagger 视觉与交互。

**Tech Stack:** FastAPI + PIL + Pydantic（后端）；Vue3 + naive-ui + vitest（前端）。

## 关键决策（用户确认）
1. 提示词收藏页形态 = **图库网格式**（上传反推→网格→点图拆分）
2. 反推来源 = **本页直接上传图片**
3. 反推图存储 = **存 promptbox 工作区**（data/promptbox/workspace/，与图库隔离，不污染）

## Global Constraints
- promptbox 与图库物理隔离（用户强调，且刚清理过图库测试图）。
- naive-ui NSelect render-label 是 **prop**（非插槽），用 `:render-label="fn"`。
- 反推图工作区复用 tagger/classifier，但存储走 PromptboxStore，不经 Storage/save_upload。
- classify 返回 dict[str, CategoryData]，PromptboxItem.categories 是 dict[str, list[str]]（无 phrase/score）。

## File Structure
